# Q34 Math Labeling MVP

This repository contains a semantic labeling prototype for a LaTeXML-generated Q34 solution. It now supports both the original four-step MVP output and a fuller Q34 step-discovery pass over the entire XML file.

The scope of this MVP:

- `S0.Ex10`
- `S0.Ex11X`
- `S0.Ex12`

From those equations, the pipeline extracts four labeled steps:

1. trig substitution: `x = a sin(theta)`
2. substitution applied inside `sqrt(a^2 - x^2)`
3. power-of-a-product rewrite for `(a sin(theta))^2`
4. algebraic simplification inside the radical

## Repository Layout

- `src/q34_mvp_pipeline.py`: thin CLI wrapper for the original four-step MVP output
- `src/semantic_labeling_pipeline.py`: generalized pipeline runner with `q34_mvp` and `q34_full` modes
- `src/q34_pipeline.py`: reusable pipeline logic for the current Q34 implementations
- `src/mathml_parser.py`: MathML-to-AST parsing helpers
- `src/step_segmentation.py`: full-Q34 step discovery and segmentation helpers
- `src/xml_loader.py`: helper for loading LaTeXML XML files
- `data/input/Q34_Version3_mathonly.xml`: exact XML input used by the MVP
- `data/output/q34_mvp_labels.json`: example output generated by the MVP
- `data/output/q34_full_labels.json`: example output from the full-Q34 discovery pass
- `docs/Q34_MVP.md`: project notes and step-by-step scope description

## Requirements

- `lxml`

## Run

From the repository root:

```bash
python3 src/q34_mvp_pipeline.py
```

That reads:

- `data/input/Q34_Version3_mathonly.xml`

and writes:

- `data/output/q34_mvp_labels.json`

To run the generalized entrypoint:

```bash
python3 src/semantic_labeling_pipeline.py --pipeline q34_full --output data/output/q34_full_labels.json
```

## What The Output Contains

The MVP JSON output includes:

- the source file path
- the selected equation scope
- four extracted step records

Each step record contains:

- `step_id`
- `source_equation_id`
- `description`
- `ast`
- `structural_labels`
- `step_labels`

The `q34_full` output uses the same core step shape, plus source provenance fields such as `source_math_id`, `raw_text`, and `cue_text` when they are available from step discovery.
