# Semantic Labeling Full Q34

## Purpose
This repository contains the full semantic labeling pipeline for Question 34. Its goal is to process the full Q34 solution, build abstract syntax tree representations of the mathematical expressions, and assign semantic labels that describe the mathematical transformations used across the solution.

Unlike the MVP repository, which focuses on only a few selected equations, this repository is designed for broader coverage of the full Q34 workflow.

## Project Structure
- `src/semantic_labeling/run.py`: main entry point for running the full Q34 semantic labeling pipeline
- `src/semantic_labeling/ast/parser.py`: builds AST-based step representations from parsed math expressions
- `src/semantic_labeling/io/xml_loader.py`: loads the LaTeXML-generated XML input
- `src/semantic_labeling/labels/definitions.py`: shared catalog of label IDs and label names
- `src/semantic_labeling/labels/matching.py`: helper logic for label matching
- `src/semantic_labeling/labels/q34_labels.py`: Q34-specific label detection logic
- `data/input/Q34_Version3_mathonly.xml`: XML input used by the full pipeline
- `data/output/q34_semantic_labels.json`: example output generated by the full pipeline
- `requirements.txt`: Python dependency list

## Requirements
- `lxml`

## How to Run
From the repository root, run:

```bash
PYTHONPATH=src python3 -m semantic_labeling.run
```

This reads:

- `data/input/Q34_Version3_mathonly.xml`

and writes:

- `data/output/q34_semantic_labels.json`

## Output Format
The JSON output contains four top-level sections:

- `metadata`
- `label_definitions`
- `labels`
- `equations`

## What the Top-Level Fields Mean

### `metadata`
This describes the overall pipeline that produced the output.

Example:
- `pipeline_name`: the name of the pipeline used to generate the file
- `focus`: the main kinds of output the pipeline is producing

Example structure:

```json
"metadata": {
  "pipeline_name": "semantic_labeling_q34",
  "focus": ["ast", "labels"]
}
```

### `label_definitions`
This is the shared label catalog used by the pipeline. Each entry defines a label ID and its human-readable name.

Example entries:
- `MAT0001` → `TRIG_SUBSTITUTION`
- `MAT0002` → `POWER_OF_A_PRODUCT`
- `MAT0003` → `TRIG_IDENTITY`

This section helps keep the label system consistent across all extracted equations.

### `labels`
This is the solution-level label list. It can be used for labels that apply to the full solution rather than only to one equation step.

At the moment, this may be empty depending on what the current pipeline detects.

### `equations`
This is the main output list. Each item represents one extracted equation or step from the Q34 solution, along with its AST and assigned semantic labels.

## What Each Equation Record Contains

Each item inside `equations` contains:

- `equation_id`
- `step_id`
- `original_expression`
- `labels`
- `ast_tree`

## What the Equation Fields Mean

### `equation_id`
This is the identifier of the equation in the original LaTeXML-generated XML.

Example:
- `S0.Ex1`

It helps trace the extracted record back to the source XML.

### `step_id`
This is the local step identifier assigned by the pipeline.

Example:
- `STEP_1`
- `STEP_2`

It is used to distinguish extracted steps inside the output file.

### `original_expression`
This is the original parsed expression string for that equation or step.

Example:
- `integral@((x ^ 2 / (4 - x ^ 2) ^ (3 / 2)) * differential-d@(x))`

It preserves the expression in a compact machine-readable form before AST formatting.

### `labels`
This contains the semantic labels assigned to that equation or step.

Examples of possible labels include:
- `TRIG_SUBSTITUTION`
- `POWER_OF_A_PRODUCT`
- `TRIG_IDENTITY`
- `FACTORING`
- `ANTIDERIVATIVE`
- `BACK_SUBSTITUTION`

Depending on the equation, this list may be empty or may contain multiple labels.

### `ast_tree`
This is the formatted abstract syntax tree for the equation.

It shows the mathematical structure line by line, with indentation representing nesting.

Example shape:

```text
int
  multiply
    divide
      power
        identifier(𝑥)
        number(2)
```

This makes the expression structure easier to inspect than raw text alone.
